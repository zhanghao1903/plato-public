# Plato 常见问题

## Plato 是什么？

Plato 是一个 Task-first 智能工作台。你输入一个目标，Plato 将它转成可见的任务计划，你可以检查、修改、确认，然后再让系统执行，并在事后查看发生了什么。

## Plato 和 ChatGPT、Claude 有什么区别？

传统聊天产品的核心对象是对话。Plato 的核心对象是任务计划。

Plato 不只是回答问题，它更强调让工作过程可见：

- 系统认为要做什么；
- 当前哪个任务被选中或正在运行；
- 哪些地方需要你的输入；
- 文件是否发生变化；
- 事后有什么证据可以查看。

## 普通用户现在可以日常使用吗？

还不建议当作正式日常工具使用。当前 `0.1.0` 是本地候选版本，适合早期评估产品方向和工作流。

它已经可以展示 Task-first 的核心体验，但仍然有一些粗糙部分，例如未签名的 macOS 安装包、持续变化的 UI 和还在增强的工作流。

## 为什么 macOS 会提示风险？

因为 `0.1.0` 版本未签名、未公证。macOS Gatekeeper 可能会阻止普通双击打开。

请参考英文文档 [macOS local release usage](../../usage/macos-local-release.md)。

## Plato 会修改我的文件吗？

Plato 的设计目标是围绕本地 workspace 和任务级控制展开。某些工作流可能会涉及文件读取或修改，但产品方向是先让任务计划可见，在需要时确认，再让用户事后检查文件变化和审计证据。

早期测试时，请优先使用非敏感的测试 workspace。

## ASK 是什么？

ASK 表示 Plato 缺少用户拥有的信息，而且不应该自己猜。

例如：

- "这个内容面向谁？"
- "是否使用默认选项继续？"
- "这个任务应该作用在哪个文件或范围？"

ASK 不等于确认。确认表示系统知道动作是什么，但需要你批准；ASK 表示系统还缺少信息。

## Audit Page 是什么？

Audit Page 是只读的信任页面。它用于查看证据、诊断信息、消息和任务事实，帮助你理解发生了什么。

Main Page 用来控制工作，Audit Page 用来建立信任。

## 数据保存在哪里？

当前公开版本是本地优先的候选版本。运行状态、日志和 workspace 数据通常保存在本机，具体取决于你选择的 workspace 和 app 配置。

在完全了解行为前，不要把密钥、客户数据、生产文件放进测试会话。

## 这个 public 仓库有源码吗？

没有。这个仓库当前用于公开产品文档、视觉资产、发布元数据和下载资产，不是源码镜像。

## 第一次应该测试什么？

建议从低风险规划任务开始：

- 规划个人网站；
- 规划课件或学习材料；
- 把一个研究想法拆成任务；
- 为小型项目整理准备步骤。

更多例子见 [产品用例](../product/use-cases.md)。

## 如何反馈问题？

可以使用 GitHub issue templates：

- Bug report：报告可复现问题；
- Product feedback：反馈产品体验和方向；
- Question：提出疑问。

不要直接上传包含隐私的日志、真实 workspace 文件或密钥信息。
