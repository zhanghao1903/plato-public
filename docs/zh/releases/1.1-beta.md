Product 1.1 的检查与使用量基础能力，便于早期评估。

## 本次新增

### Token Usage Analytics

- 增加本地 token 使用记录与聚合基础。
- 支持从 workspace / session 层面理解 AI runtime 的使用量和 cache 行为。
- 当前 beta 重点是可见性；计费和强预算控制不在本次范围内。
- 相关文档：[英文 User guide](../../usage/user-guide.md#5-inspect-the-result)、
  [隐私与安全](../security/privacy-and-safety.md)。

### Precision File Tools

- 增加精确文件操作基础：按行读取、workspace 搜索、带 hash 保护的行范围替换、
  幂等 append，以及 changed-line evidence。
- 让 AI 辅助文件修改更小、更可审查，避免大范围不透明改动。
- 为文件操作提供更强的审计证据。
- 相关文档：[英文 Trust and audit](../../architecture/trust-and-audit.md#evidence-anchors)、
  [隐私与安全](../security/privacy-and-safety.md)。

### Workspace Inspection: Git / Diff / File Viewer

- 增加 workspace inspection：Git 状态、结构化 diff、文件查看路径。
- 使用 renderer-safe 的 workspace 和路径标签，避免在公开 UI 中暴露本地原始路径。
- 用户可以从任务结果进入变更文件和证据，而不需要离开本地应用上下文。
- 相关文档：[英文 Trust and audit](../../architecture/trust-and-audit.md#workspace-inspection-direction)、
  [Workspace inspection 截图](../../../assets/screenshots/plato-workspace-inspection.png)。

## 发布验证

本次 release asset refresh 基于私有仓库 `main` commit
`8011bd5fa7244199311b033b15152a93b6712e37` 重新构建。

发布前已验证：

- frontend runtime、smoke、Electron sidecar focused tests；
- Python multi-workspace 与 workspace inspection contract tests；
- launcher-backed Electron smoke：configured、first-run、startup diagnostics；
- `hdiutil verify` 验证最终 DMG；
- 挂载最终 DMG 后的 installer smoke：configured、first-run、startup diagnostics。
- 使用 `--release-version 1.1-beta` 验证显式发布版本打包，版本元数据由
  packaging 脚本生成，不再手动修改生成文件。

## 已知限制

- 未签名；
- 未公证；
- macOS Gatekeeper 可能拦截普通打开方式；
- 仅提供 macOS Apple Silicon 版本；
- 本仓库不是源码镜像；
- 部分 Product 1.1 能力仍是基础版：自动化策略、强预算控制、市场化分发不在本次范围内。

## 推荐第一次测试

1. 在测试 workspace 中打开 Plato。
2. 准备一个已初始化 Git 的本地项目。
3. 启动低风险任务并检查可见任务计划。
4. 打开 workspace inspection，查看 Git 状态、变更文件和 diff。
5. 有活动后打开 Audit Page 检查证据。
6. 在 AI 活动后查看 token 使用摘要。
