# 隐私与安全

这份文档用普通语言说明当前公开版本的安全边界。

Plato `1.1-beta` 是早期本地 beta 版本。它适合评估，但不是经过正式安全审计或应用商店分发的产品。

## 本地优先评估

当前公开版本是本地桌面 App，包含 bundled sidecar runtime candidate。

建议：

- 先使用非敏感测试 workspace；
- 在了解行为前，不要放入密钥、凭证、客户数据或生产文件；
- 信任结果前，先查看任务状态、文件变化和 Audit Page。

## 用户控制

Plato 的产品方向是让用户保持控制：

- 目标会转成可见任务计划；
- 重要执行前可以检查任务；
- 缺少用户信息时出现 ASK；
- 高风险动作应该就近确认；
- Audit Page 是只读证据页面。

这不表示当前版本解决了所有风险，而是表示产品设计方向是可见控制，而不是隐藏自动化。

## 文件和 Workspace

某些 Plato 工作流可能会检查或修改本地 workspace 文件。

使用真实项目前建议：

1. 使用版本控制；
2. 风险较高时使用临时分支或副本；
3. 发布前检查任务；
4. 执行后检查文件变化和审计证据。

## 日志与审计

Plato 使用日志、消息、任务状态和审计投影帮助解释发生了什么。

不要公开分享未经检查的原始日志。日志可能包含本地路径、prompt、文件名或其他私有上下文。

## 网络和 Provider

公开文档描述的是 Plato 的产品和本地 release 姿态。实际模型/provider 行为取决于运行环境配置。

不要把 provider secret 或 API key 放到公开 issue、截图或日志中。

## 当前分发限制

`1.1-beta` 版本：

- 未签名；
- 未公证；
- 用于本地评估；
- 不是正式安全版本；
- 不承诺每张截图展示的能力都已经稳定发布。

具体见英文 [Release status](../../product/release-status.md)。

## 如何反馈安全相关问题

可以用 GitHub issue 提交公开安全的描述。如果问题需要敏感细节，请先用高层描述，避免直接上传私有日志或 workspace。

有效反馈通常包括：

- release version；
- 操作系统和架构；
- 预期行为；
- 实际行为；
- 是否有文件变化；
- 已脱敏截图或日志。
